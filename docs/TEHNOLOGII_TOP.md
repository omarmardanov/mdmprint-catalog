# Верхний блок «Технологии печати» / «Каталог материалов» — табы + плитки

Прототип: `prototypes/tehnologii-top.html`. Живые страницы:
`mdmprint.ru/tehnologii-pchati` и `mdmprint.ru/katalog-materialov`.

## Что делаем

На странице сейчас вверху **три плитки-ссылки** на группы. Заменяем их на **табы групп + плитки
услуг** под активным табом. Клик по плитке ведёт сразу на страницу услуги (уже свёрстанную), без
захода на промежуточную страницу группы.

**Меняем ТОЛЬКО табы и плитки.** Не трогаем:
- хлебные крошки и заголовок H1 (остаются существующие);
- всё, что ниже блока (Требования к файлам, Портфолио, Отзывы, FAQ, форма).

## КРИТИЧНО для SEO — не осиротить страницы групп

Табы уводят человека сразу на услугу, минуя страницы групп (`/broshyurovka-i-pereplyot`,
`/postpechatnaya-obrabotka`, `/pechat`). **Эти страницы удалять и отвязывать НЕЛЬЗЯ** — они
приносят поисковый трафик по групповым запросам.

Поэтому в каждом табе есть ссылка **«Все услуги группы →»** на страницу группы. Она:
- сохраняет внутреннюю ссылку на страницу группы (SEO цела);
- даёт путь тем, кому нужен обзор группы целиком.

Это требование к дизайну, а не опция. Подтверждено как решение с Omar; при сомнениях — сверить
с сеошником заказчика.

## Страница группы (промежуточная) — та же плитка, без табов

Страницы групп из блока выше (`/broshyurovka-i-pereplyot` и др.) не только сохраняем ради SEO —
у них свой простой шаблон: **H1 группы + вводный абзац + плитки услуг этой группы**. Табы не нужны
(группа уже одна), в остальном — тот же компонент, что в верхнем блоке:

- плитки — `.mk-tilelink`, раскладка `combo-grid` (3 в ряд на десктопе, 1 на мобилке);
- каждая плитка ведёт на страницу услуги; состав и порядок — по таксономии группы;
- **отдельного прототипа нет намеренно** — это верхний блок минус табы, копировать нечего.

Иерархия каталога целиком: **страница «Технологии» (табы) → страница группы (плитки) → страница
услуги** (эталон `usluga.html`). Средний уровень — этот, лёгкий.

## Интеграция

| Пункт | Значение |
|---|---|
| Ширина контейнера | **1200px** — под живой `.container_small` (замерено в девтулзах). Наш стандартный 1176 не использовать здесь. |
| Фон | **белый** (`--surface-page`), как на старой странице. Своей секции-подложки у блока нет. |
| Табы | client-side фильтр, без перезагрузки. Активный таб — подчёркивание брендом. |
| Плитки | компонент кита **`.mk-tilelink`** (кадр слева, заголовок+подпись, стрелка). Раскладка — `combo-grid`, 3 в ряд на десктопе, 1 на мобилке. |
| CSS | наши `tokens.css` + `kit.css`. **Следить за специфичностью против CSS темы** (см. `HANDOFF.md` §5) — тема WordPress добавит свой сброс/каскад. |

## Данные / CMS

- **Таб** = группа услуг (существующая таксономия групп). На Технологиях их 3.
- **Плитка** = услуга группы: заголовок, короткая подпись (≤ ~30 знаков), картинка-кадр, URL страницы услуги.
- **Ссылка «Все услуги группы»** = URL страницы группы.
- Материалы (`katalog-materialov`) — **тот же компонент**, табы по типам материалов. Списки свои.

## Заглушки в прототипе (заменить реальным)

- **Списки услуг для «Постпечатной обработки» и «Печати» — рыба.** Реальные списки подтверждает
  маркетолог/менеджер. Группа «Брошюровка и переплёт» — реальные услуги и кадры.
- Кадры для части плиток «Постпечатной» и всех «Печати» — плейсхолдеры («кадр»), нужны реальные `.webp`.
- Крошки и заголовок в прототипе показаны бледным как контекст — на живой странице берутся существующие.
