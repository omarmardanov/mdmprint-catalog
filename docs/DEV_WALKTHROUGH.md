## Этап 0. Открыл файл

### ✅ 0.1. Три класса на `<body>` — ИСПРАВЛЕНО 17.07

```html
<body class="fcopy-hint shape-rounded v-after">
```

Это **замороженные тумблеры прототипа**, оставшиеся от сравнения вариантов. Их не видно в макете, но:

| Класс | Что делает | Упоминаний в CSS/JS |
|---|---|---|
| `v-after` | вариант «после редизайна» — от него зависит половина раскладки | **71** |
| `shape-rounded` | форма: радиус 8 вместо пилюль | 9 |
| `fcopy-hint` | вариант Б копирования телефона (подсказка вместо кнопок) | 15 |

**Разработчик их не поставит** — в теме WordPress `<body>` генерирует WP, и класс `v-after` там
взяться неоткуда. Страница молча поедет: правила `body.v-after .X` просто перестанут применяться, а
это самые специфичные правила в файле.

**Сделано:** запечены. Вариант «после» стал единственным: удалены 6 svg варианта «до», снят класс
`v-after-el`, 78 селекторов лишились префикса, `<body>` без классов, из JS убрана проверка
`fcopy-hint`. Доказано слепком вычисленных стилей до/после (699 и 745 элементов × 23 свойства на
двух вьюпортах): вёрстка не изменилась. Коммит `787f578`.

### ✅ 1.2. Промо-блок в мегаменю: делаем ОДИН вариант (решено 23.07)

В коде `data-variant`: `offer`, `banner`/`image`, `content`, `hybrid`. Это один компонент с четырьмя
состояниями — их показывали заказчику на созвоне, отсюда четвёрка.

**К реализации — только `hybrid`,** тот, что стоит в прототипе по умолчанию. Поля выбора варианта в
админке нет и не заводим: переключатель без спроса усложняет и админку, и вёрстку, а сезонная
подмена промо пока никем не запрошена. Остальные три остаются **описанием на будущее** — разметка
в прототипе лежит, но в тему не переносится.

Если вариант понадобится — это отдельная задача: добавить поле-селектор и перенести три готовых
состояния. Разметку из прототипа при этом не выбрасывать.

## Этап 3. Шаблон страницы услуги

### 🟡 3.1. Подшапка: якоря захардкожены

Пункты ведут на `#obzor`, `#preim`, `#vidy`, `#ceny`, `#primery`, `#trebovaniya`, `#faq`… Секции
включаются по данным (см. `CMS_FIELDS.md`), значит **подшапка обязана собираться из включённых
блоков**, иначе пункт поведёт в пустоту. В прототипе это статичный список — переписывать в шаблоне.

### 🟡 3.2. Скроллспай, ленты, табы — на `querySelectorAll` по всей странице

JS ищет элементы глобально (`[data-tabs]`, `.mk-scroller`, `.usl-subnav a`). Если тема добавит
второй такой блок (или виджет), поведение поедет. **Вопрос:** страница у нас одна на шаблон или
блоки могут повторяться?

### ✅ 3.3. Кадр героя — ИСПРАВЛЕНО 17.07

Был PNG **1374 КБ** в корне ассетов под именем `image.png` — и это LCP-элемент страницы.
Стал `assets/img/hero/tisnenie-vizitka.webp` — **53 КБ, минус 96%** при том же разрешении, плюс
`width`/`height` (CLS) и `fetchpriority="high"`, без `lazy` (первый экран).

Заодно проверено то, чем пугает наше же правило ДС («прозрачный PNG + `mix-blend-mode:darken` = чёрные
пятна»): у кадра 0,1% полупрозрачных пикселей и **ноль тёмных среди них** — ореола нет. Правило
остаётся предупреждением, но этот кадр безопасен.

### 🟡 3.4. Цены вбиты руками

Три строки в HTML. Источник — ХП API (`QUESTIONS.md` §1). До ответов ХП блок не переносится в

## Этап 5. Общее по каскаду

### 🔴 5.1. Специфичность: тема добавит свой CSS

В прототипе уже **шесть** мест, где правило написано и молча не работает: `.usl section` (0,1,1)
бьёт `.usl-price` (0,1,0); `:not([class])` даёт +1; `body.v-after .X` из более позднего `<style>`
бьёт дважды. В теме добавится её собственный CSS — и таких мест станет больше.

**Требование:** порядок подключения `tokens → kit → шаблон`, никакой минификации в общий бандл с
чужими стилями. Проверять **замером** (`getComputedStyle`), а не глазами.

### ⚪ 5.2. Два мобильных брейкпоинта в одном файле

В прототипе `@media(max-width:900px)` встречается **дважды**: в основном `<style>` и в
`<style id="port-after">`. Правило для селектора с `body.v-after`-двойником обязано жить во втором,
иначе проигрывает и специфичностью, и порядком. При переносе это надо **схлопнуть в один блок** —
иначе разработчик будет ловить призраков.

### ⚪ 5.3. Кэш CSS

Ссылки на CSS обязаны идти с `?v=`. Без него браузер отдаёт старый файл часами — 16.07 мы на этом
потеряли время: телефон показывал новый HTML со старым CSS, и это выглядело как «шрифты выросли».
