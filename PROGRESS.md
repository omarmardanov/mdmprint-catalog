## UX-беклог (юзабилити-полировка) — накапливаем, пройдёмся пачкой
Микровзаимодействия / эвристики (Нильсен) — правим отдельным слоем поверх вёрстки, разом по всем страницам и формам, а не по одному.

- [ ] **Валидация форм: превентивный `disabled` → реактивный.** Кнопку держать активной, по клику подсвечивать незаполненные поля + инлайн-ошибки (best practice GOV.UK / NN-g, лучше для a11y и screen-reader). Касается ВСЕХ форм: `cbmBack` (звонок), `reqBack` (заявка), FAQ-форма (`.faq-form`). Общий обработчик — `setupLead` + отдельный у FAQ. Omar склоняется к реактивному; статус-кво пока — превентивная блокировка. Контекст — память `header-catalog-prototype-state`.
- [ ] **Exclusive-аккордеон (single-open).** Открыт только один пункт; открытие другого схлопывает предыдущий. Без JS — атрибут `name="…"` (одинаковый на группе `<details>`, baseline браузеров с 2024). Где есть аккордеоны: FAQ на `prototypes/usluga.html` (`.mk-acc`).

_Как пользоваться: замеченную юзабилити-мелочь дописывать сюда чекбоксом, не внедряя сразу; проходим списком отдельной сессией._

## Промоблок справа — переделан в переключаемый компонент (как в индустрии)
- Принцип (Shopify/WB/OZON): ОДИН компонент .promo + модификатор data-variant, слоты ОПЦИОНАЛЬНЫ (нет в разметке → не рендерится). Маркетолог в CMS выберет «тип баннера» и заполнит поля
- 4 типа (тела .promo-body[data-for], показывается активное через CSS-селекторы по data-variant):
  - **offer** — бейдж+заголовок+текст+SVG-мокап стопки визиток+цена+кнопка. Мокап заполняет бывшую пустоту, цена/кнопка прижаты вниз (flex:1 на .promo-visual)
  - **image** — только кликабельный баннер (.banner-fill градиент + .banner-cap подпись)
  - **content** — обложка сверху (.promo-cover) + бейдж + заголовок + текст + ссылка «Читать» (margin-top:auto)
  - **hybrid** — тёмный фон + жёлтое свечение (::before radial) + бейдж + текст поверх + белая ghost-кнопка
- Демо-переключатель .promo-switch (Оффер/Баннер/Контент/Гибрид) вверху промо + JS в конце script. В проде это будет селект в CMS, переключатель убрать
- Все мокапы — inline SVG, без внешних файлов
- Починен баг: случайная стрелка .ha у «₽» в заголовке промо (остаток perl-скрипта для h4) — убрана
- ВАЖНО при рендере вариантов: sed заменять 'id="promo" data-variant="offer"' (не просто data-variant — иначе ломает CSS-селекторы)

**Специфичность в `usluga.html` бьёт кит.** Три случая: `.usl p:not([class])` (0,2,1) перекрасил абзац
CTA; `.usl-foot a{display:block}` (0,1,1) сломал `display` у `.mk-btn`; ранее `.mk-btn{display:inline-flex}`
перебивал `[hidden]`. Проверять computed-стиль, а не глаз. И не резать CSS «от маркера до маркера» —
так я снёс правила `.usl-final`/`.usl-foot` вместе с блоком статей.

### Специфичность: четвёртый и пятый случай
`.usl-foot .flogo{display:block}` и `.usl-foot .fbadge{display:inline-flex}` имеют **ту же**

### «С чем сочетается»: darken на светлой подложке
`.usl-combo .mk-tilelink__media img{mix-blend-mode:darken}` — зеркало приёма из CTA.

плитки (`--surface-gray`), белый прямоугольник и шов исчезли. Правки scoped под `.usl-combo`.

срез < 1px). Временно рядом стоит демо-плитка «Конгрев (старый кадр)» (`kongrev2.webp`) —

### Шапка: два бага мегаменю
- **Подсветка в навбаре при открытии каталога.** `showSec()` красила быстрые ссылки заодно

  вероятен кэш; ждём уточнения, не про затемнение ли навбара речь.)
- **Толстая линия между меню и шапкой.** Смещения абсолютного блока считаются от **padding-box**:

  `body.scrolled` — `top:100%` (у навбара границы нет, иначе белая щель). Замерено: ровно одна
  полоса `#e4e4e4` в 1px в обоих состояниях.
